# Ход выполнения проектной практики

## Введение

Проектная практика выполняется на основе проекта STEAMLEAF.

STEAMLEAF — это разрабатываемая сюжетная игра в жанре нарративного квеста с элементами RPG. Игрок берёт на себя роль промоутера, который раздаёт листовки в стимпанк-городе, взаимодействует с персонажами, выполняет квесты и влияет на социальный рейтинг.

В рамках практики разрабатываются:

- сайт проекта;
- Telegram-бот по игре;
- документация;
- итоговый отчёт.

## Цель практики

Целью проектной практики является создание информационного сайта и интерактивного Telegram-бота для презентации игры STEAMLEAF.

## Задачи практики

Для достижения цели были поставлены следующие задачи:

- изучить требования к проектной практике;
- создать Git-репозиторий;
- подготовить структуру репозитория;
- оформить документацию в формате Markdown;
- собрать материалы по игре STEAMLEAF;
- разработать статический сайт;
- разработать Telegram-бота;
- подготовить итоговый отчёт.

## Этап 1. Изучение задания

На первом этапе были изучены требования к проектной практике.

Было определено, что репозиторий должен содержать следующие основные разделы:

- `docs/` — документация;
- `reports/` — отчёты;
- `site/` — сайт проекта;
- `src/` — исходный код;
- `task/` — задание и шаблон отчёта.

## Этап 2. Создание Git-репозитория

Для проекта был создан репозиторий `steamleaf-practice` на GitHub.

В репозитории были подготовлены основные папки:

- `docs/`;
- `reports/`;
- `site/`;
- `src/`;
- `task/`.

Также были созданы базовые файлы:

- `README.md`;
- `.gitignore`.

## Этап 3. Подготовка материалов проекта

Для проекта STEAMLEAF были собраны и систематизированы материалы:

- описание игры;
- описание мира;
- описание персонажей;
- описание районов;
- листовки;
- скриншоты интерфейса;
- визуальные материалы;
- идеи для Telegram-бота.

## Этап 4. Разработка сайта

Сайт создаётся в папке `site/`.

Сайт должен представить игру STEAMLEAF и содержать основные разделы:

- главная страница;
- описание проекта;
- мир игры;
- персонажи;
- листовки;
- журнал разработки;
- Telegram-бот;
- ресурсы.

Сайт разрабатывается с использованием HTML и CSS.

## Этап 5. Разработка Telegram-бота

Telegram-бот создаётся как интерактивное дополнение к игре.

Бот должен знакомить пользователя с миром STEAMLEAF, персонажами, районами, листовками и системой социального рейтинга SR.

Планируемые функции Telegram-бота:

- главное меню;
- раздел «Об игре»;
- раздел «Персонажи»;
- раздел «Районы»;
- раздел «Листовки»;
- мини-игра «Рабочий день промоутера»;
- отображение социального рейтинга SR;
- ссылка на сайт проекта.

## Этап 6. Подготовка отчёта

После завершения разработки сайта и Telegram-бота будет подготовлен итоговый отчёт.

Отчёт будет размещён в папке `reports/` в двух форматах:

- DOCX;
- PDF.

## Промежуточный результат

На текущем этапе:

- создан Git-репозиторий;
- подготовлена структура проекта;
- начато оформление документации;
- определена концепция сайта;
- определена концепция Telegram-бота.

## Итоговый результат

Итогом проектной практики должны стать:

- заполненный Git-репозиторий;
- сайт проекта STEAMLEAF;
- Telegram-бот по игре;
- документация;
- итоговый отчёт.
